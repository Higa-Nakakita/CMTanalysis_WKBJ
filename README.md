# CMTanalysis_WKBJ

WKBJで必要となるtxtファイルdiv_del_infoを作成するためのコード群．WKBJコードで参照するローカルモードの座標決定と線積分の実行（各グリッドの波線通過距離等）で必要となる．
src_coordinatesに震源位置を入力のち，mk_divdel.outを実行すると，div_del_infoが作成される．なお日本を分割するグリッドは，ローカルモードを保持するそれと対応していなければならない．

ファイル説明

gcp_grids.py
交点座標と波線通過グリッドの計算，出力．アウトプットファイルはfil = 'output_'+f'{slat_str}_{slon_str}'+'.txt'．
各波線とグリッド（日本周辺を0.5deg分割）との交点座標２つと波線が通過したグリッドを指定する座標（通過グリッドの端西東南北）を求める．加えて，各交点までの震央距離をobspyで計算し，逐次保存する．現状，指定した震源位置からFーnet全点までの，交点座標と通過グリッドを求める仕様になっている．

Makefile
gfortranコンパイラーを使用．


mk_divdel_info.f90
gcp_grids.pyの出力結果を読み込み，各グリッドの通過距離や各交点までの震央距離を，fortranコードazdelで再度計算．計算結果がobspyによるものと大きく異なる場合（5km以上）処理を停止（震央距離を3000kmくらいに広げると大体止まる．未解決だが日本の地震をF-netでやる分には問題ない）．
